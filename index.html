Computer Security Concepts

What is Computer Security?

  • It refers to techniques for ensuring data stored in a computer cannot be read or compromised by any individuals without authorization.
  • It involves encryption and password.

    • Security Terminology

      1. ADVERSARY (THREAT AGENT)

      An entity that attacks or is threat to a system.

      2. ATTACK

      An assault on system security that derives from an intelligent data.

      A deliberate attempt to evade security services and violate security policy of a system.

      3. COUNTER MEASURE

      An action or technique that reduces threat, a vulnerability or an attack by eliminating or preventing it, by minimizing the harm it can cause.

      4. RISK

      An expectation of loss expressed that a particular threat will exploit a particular vulnerability with a particular harmful result.

      5. SECURITY POLICY

      Set of rules and practices that specifies how a system or an organization provides security services to protect sensitive and critical system resource.

      6. THREAT

      A potential violation of security, which exits when there is a circumstance, capability, action or event that could breach security and cause harm.

      7. VULNERABILITY

      Flaw or weakness in a system design, implementation or operation and management that could be exploited to violate the systems security policy.

    Vulnerability

    System resource vulnerabilities may be:

      Corrupted

      Does the wrong thing or gives wrong answer.

      Leaky

      Someone who should not have access to the information will avail.

      Unavailable

      Otherwise very slow, using the system network is impossible

    Types of Vulnerability

      1. HARDWARE VULNERABILITIES

      Attack by adding devices, changing or removing them, intercepting the traffic from them, or flooding them with traffic until they are no longer function.

      2. SOFTWARE VULNERABILITIES

      Softwares can be replaced, changed or destroyed maliciously, or it can be modified, deleted or misplaced accidentally whether intentional or not.

      3. DATA VULNERABILITIES

      Data have definite value, even though that value is often difficult to measure.

    Threats

      1. INTERCEPTION

      Unauthorized party has gained access to an asset. The outside party can be a person, program or computing system.

      2. INTERRUPTION

      An asset of the system may be lost, unavailable or unusable.

      3. MODIFICATION

      Unauthorized party tampers with an asset.

      4. FABRICATION

      Counterfeit objects on a system and intruder may insert spurious transaction to a network communication system or add records to an existing database.

    Classification of Attacks

      1. INSIDE ATTACK

      Initiated by an entity inside the security perimeter.

      2. OUTSIDE ATTACK

      Initiated from outside the perimeter by an unauthorized or illegitimate user of the system.

    Types of Attacks

      1. ACTIVE ATTACK

      Attempts to alter system resources or affect their operation

    • Replay, Masquerade, Modification of Messages, Denial of Service
      2. PASSIVE ATTACK

      Attempts to learn or make use of information from the system but does not affect system resources.

    • Release of message contents, Traffic Analysis

    Attack Surfaces

    Consist of the reachable and exploitable vulnerabilities in a system.

      1. Network Attack Surface

      Vulnerabilities over an enterprise network, wide area network or the internet.

      2. Software Attack Software

      Vulnerabilities in the application utility or operating system code.

      3. Human Attack Surface

      Vulnerabilities created by personnel or outsiders, such as social engineering, human error and trusted insiders.

    Unauthorized Disclosure

    A threat to confidentiality, it results in various attacks like:

      1. Exposure

      Sensitive data is directly released to unauthorized entity.

      2. Interception

      An unauthorized entity directly accesses sensitive data in transit.

      3. Inference

      An unauthorized entity indirectly accesses sensitive data.

      4. Intrusion

      An unauthorized entity circumvents system security protection.

      3. Human Attack Surface

      Vulnerabilities created by personnel or outsiders, such as social engineering, human error and trusted insiders.

    Deception

    Is a threat to either system integrity or data integrity; it results in various attacks like:

      1. Masquerade

      An unauthorized entity poses an authorized entity.

      2. Falsification

      False data deceives an authorized entity.

      3. Repudiation

      An entity deceives another by falsely denying responsibility for an act.

    Countermeasure

    It can be devised to the following:

  • 1. Prevent a particular type of attack
  • 2. Detect the attacker and the type of attack
  • 3. Recover from the effects of the attack
    • Tenets of Information System Security

    CONFIDENTIALITY

    Guarding information from everyone except those with rights to it. Only authorized users can view the information.

    Confidential information includes the following:

  • 1. Private data of individuals
  • 2. Intellectual property of businesses
  • 3. National security for countries and governments

    • Security Control Examples:

      1. Conduct annual security awareness

      This helps remind about proper handling of private data. It drives awareness of the organization’s framework of security policies, standards, procedures and guidelines.

      2. Create IT security policy framework in place

      A policy framework is like an outline that identifies where security controls should be used.

      3. Designing a layered security solution for an IT Infrastructure

      The more layers or compartments that block or protect private data and intellectual property, the more difficult the data property are to find and steal.

      4. Performing periodic security risk assessments, audits, and penetration test on websites and IT infrastructure

      This is how security professionals verify that they have properly installed the controls.

      5. Enabling security incident and event monitoring at your internet entry and exit points

      This is like using a microscope to see what is coming in and going out.

      6. Using automated workstation and server antivirus and malicious software protection

      This is the way to keep viruses and malicious software out of your computer.

      7. Using more stringent access controls beyond a logon ID and password for sensitive systems, application and data.

      Logon IDs with passwords are only one check of the user. Access to more sensitive systems should have a second test to confirm the users identity.

      8. Minimizing software weaknesses in your computers and servers by updating them with patches and security fixes.

      This is the way to keep your operating system and application software up to date.

    INTEGRITY

  • Only authorized users can change information. It deals with the validity and accuracy of data.
  • Data lacking integrity that is, data that are not accurate or not valid are of no use.
    Examples: Copyrights, Patents, Secret Formulas, Customer Databases

    • AVAILABILITY

  • Information is accessible by authorized users whenever they request the information.
  • Expressed as the amount of time users can used the system, application and data.

    • COMMON AVAILABILITY TIME MEASUREMENTS

      1. Uptime

      The total amount of time that a system, application and data are accessible. Uptime is typically measured in units of seconds, minutes and hours within a given calendar month. Often uptime is expressed as a percentage of time available. E.g. 99.5 percent uptime.

      2. Downtime

      It is the total amount of time that a system, application and data are not accessible. Downtime also is measured in units of seconds, minutes and hours for a calendar month.

      3. Availability

      The probability that a system will work as required when required during the period of a mission

      Availability is a mathematical calculation where A=(Total Uptime) / (Total Uptime + Total Downtime)

      4. Mean time to failure (MTTF)

      MTTF is the average amount of time between failures for a particular system during normal system operation.

      MTTF = Total hours of operation ÷ Total assets in use

      MTTF = 10,000 hours ÷ 40 assets

      MTTF = 250 hours

      5. Mean time to repair (MTTR)

      MTTR is the average amount of time it takes to repair a system, application or component. The goal is to bring the system back-up quickly.

      6. Mean time between failures (MTBF)

      MTBF is the predicted amount of time between failures of an IT system during operation.

      7. Recovery Time Objective

      The amount of time it takes to recover and make a system, application and data available for use after an outage.

    Seven Domains of Typical IT Infrastructure

    1. USER DOMAIN

    This Domain is where you find an Acceptable Use Policy, it defines what users are allowed and not allowed to do with organization owned IT Assets

    2. WORKSTATION DOMAIN

    A workstation can be a desktop computer, laptop computer, a special purpose terminal or ant device that connects to your network.

    3. LAN DOMAIN

    Local Area Network (LAN) is a collection of computers connected to one another or to a common connection medium.

    Physical Part of LAN Domain

      A. Network Interface Card

      The interface between the computer and the LAN physical media. The NIC has a 6 byte Media Access Control (MAC) layer address that serves as the NICs unique hardware identifier

      B. Ethernet LAN

      This is a LAN solution based on the IEEE 802.3 CSMA/CD standard for 10/100/1000Mbps. Ethernet is the most popular LAN Standard.

      C. Unshielded twisted-pair cabling

      This is the workstation cabling that uses RJ-45 connectors and jacks to physically connect to a 100Mbps/1Gbps/10Gbps. Most organization use CAT 5 and CAT 6 UTP transmission media to support high speed data communications.

      D. LAN Switch

      This is the device that connects workstation into physical Ethernet LAN. A switch provides dedicated Ethernet LAN connectivity for workstations and servers providing maximum throughput and performance.

      E. File Server and Print Server

      These are high powered computers that provide file sharing and data storage for users within a department. Print servers support shared printer use within a department.

      F. Wireless Access Point (WAP)

      For wireless LANs (WLANs), radio transceivers are used to transmit IP packets from a WLAN NIC to WAP. The WAP transmits WLAN signals for mobile laptops to connects. The WAP connects back to the LAN switch using the UTP Cables.

    Logical Part of LAN Domain

      A. System Administration

      Setup of user LAN accounts with logon ID and password access controls.

      B. Design of directory and file services

      The servers, directories and folders to which the user can gain access.

      C. Configuration of workstation and server TCP/IP software and communication protocols

      This involves IP addressing, the IP default gateway router, subnet mask address, etc. The IP default gateway router acts as the entry/exit to the LAN.

      D. Design of server disk storage space; backup and recovery user data

      Provision for user data files on LAN disk storage areas where data are backed up and archived daily. If data loss or corruption, data files can be recovered from backup files.

      E. Design of virtual LAN (VLANs)

      With Layer 2 and Layer 3 LAN switches, Ethernet ports can be configured to be on the same VLAN, even though they may be connected to different physically connected LANs.

    4. WAN DOMAIN

    This domain connects remote location. As network cost drop, organization can afford faster internet and WAN connection. Most telecommunication today provide the following service:

      a. Nationwide Optical Backbones (VLANs)

      Optical backbone trunks for private optical backbone networks.

      b. End-to-end IP Transport

      IP services and connectivity using the service providers IP networking infrastructure.

      c. Multi-site WAN cloud services

      IP services and connectivity offered for multi-site connectivity such as multi-protocol label switching (MPLS) WAN services. MPLS uses labels or tags to make virtual connections between endpoints in a WAN.

      d. Metropolitan Ethernet LAN connectivity

      Ethernet LAN connectivity offered within a city’s area network.

      e. Dedicated Internet Access

      A broadband Internet communication link usually shared among an organization.

      f. Managed Services

      Router management and security appliance management 24 x 7 x 365.

    5. LAN to WAN DOMAIN

    The LAN to WAN domain is where the IT Infrastructure links to a wide area network and the Internet. The Internet is open, public and easily accessible by anyone.

    Network applications uses two common transport protocols such as Transmission Control Protocol (TCP) and User Datagram Protocol (UDP). Both TCP and UDP use port numbers to identify application or function.

    Examples of common TCP and UDP port numbers:

      a. Port 80: Hyper Text Transfer Protocol (HTTP)

      The communication protocol between web browsers and websites with data in cleartext.

      b. Port 20: File Transfer Protocol

      A protocol for performing file transfers. FTP uses TCP as a connection oriented data transmission but in cleartext. Connection oriented means individuals packets are numbered and acknowledged as being received to increase integrity of the file transfer.

      c. Port 69: Trivial File Transfer Protocol

      A protocol for performing file transfer. TFTP utilized UDP as a connectionless data transmission but in cleartext. This is used for small and quick file transfers given that it does not guarantee individual packet delivery.

      d. Port 23: Terminal (Telnet)

      A network protocol for performing remote terminal access to another device. Telnet uses TCP and sends data in cleartext.

      e. Port 22: Secure Shell

      This is a network protocol for performing remote terminal access to another device. SSH encrypts the data transmission for maintaining confidentiality of communications.

    6. REMOTE ACCESS DOMAIN

    Remote access is critical for staff members who work in the field or from home.

    Global access makes it easy to connect to the Internet, email and other business applications anywhere your can find a Wireless Fidelity (WIFI) hotspot.

    This domain is important, but dangerous to use. It introduces many risk and threats from the internet.

    Most of the workers depend on the following:

      a. Highly available cellphone service

      Mobile workers need cellphone service to get in touch with office and support teams.

      b. Real-time access for critical communication

      Use of text messaging or IM chat on cell phones provides quick answers to short questions and does not require users to completely interrupt what they are doing.

      c. Access to email from mobile device

      Integration of email with cellphones, smartphones, personal data assistants (PDAs) or blackberry devices provides quick response to important email messages.

      d. Broadband WiFi Internet access

      Some nationwide service providers now offer WiFi broadband access cards. They allow wireless access in major metro areas.

      e. Local WiFi hotspot

      WiFi hotspot are abundant, including airports, libraries, coffee shops and retilers. While most are free, some require that users pay for access.

      f. Broadband Internet access to home office

      Staffers who work from home require broadband internet access. This is usually bundled with VOIP telephone service and digital TV service.

      g. Secure remote access to a company’s IT infrastructure

      Remote workers require secure VPN tunnels to encrypt all IP data transmission through the public Internet. This is critical if private data is being accessed remotely.

    Remote access security controls must use the following:

      1. Identification

      The process of providing identifying information such as a username, a logon ID or an account number.

      2. Authentication

      The process for proving that a remote user is who the user claims to be. The most common authentication method is supplying a password. A token can be hardware or software.

      3. Authorization

      The process of granting rights to use an organization’s IT assets, systems, application, and data to a specific user.

      4. Accountability

      The process of recording user actions. The recorded information is often used to link users to system events.

    7. SYSTEM AND APPLICATION DOMAIN

    This domain holds all the mission control critical systems, applications, and data. Authorized users may have access to many components in this domain.

    It consists of hardware and operating software, applications and data. It also includes hardware and their logical deign. It must be secured both physically and logically.

      The responsibility for Systems/Application Domain lies with the director of systems and applications and the director of software development which includes the following:

      Server Systems Administration

      Database design and management

      Designing access rights to systems and applications

      Software Development

      Software Development Project Management

      Software Coding and Testing

      Quality Assurance and Production Support
